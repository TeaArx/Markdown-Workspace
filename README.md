# Markdown Workspace

Markdown Workspace is a desktop Markdown editor built with Electron, Vue, Pinia, Markdown-it, Highlight.js and Mermaid. The app supports local file editing, live preview, notes, project charts, notifications and persistent settings.

## Features

- Open, edit, drag and save `.md`, `.markdown` and `.txt` files.
- Live Markdown preview with syntax highlighting and Mermaid diagrams.
- Local notes stored in the Electron `userData` directory.
- Chart editor that generates Markdown and Mermaid blocks.
- Light, dark and system themes.
- Autosave and focus timer notifications.
- Secure IPC through `contextBridge`, context isolation and renderer sandboxing.
- Windows NSIS installer and GitHub Releases auto-update flow.

## Requirements

- Node.js 20 or newer.
- npm.
- Windows for the configured NSIS release target.

## Development

```bash
npm ci
npm start
```

`npm start` runs `electron-vite dev`: the main and preload bundles are built, the renderer starts on a Vite dev server, and Electron is launched with hot reload for the renderer.

```bash

```

Creates the Windows NSIS installer, blockmap and update metadata in `release/` without publishing.

## Release And Updates

```bash
npm run release
```

Builds the app and publishes release assets to GitHub Releases through `electron-builder`. Publishing requires `GH_TOKEN` or `GITHUB_TOKEN`.

Auto-updates are enabled only in a packaged app. For Windows, the configured flow uses:

- `electron-builder` with the NSIS target.
- `electron-updater` in the Electron main process.
- GitHub Releases as the update provider.
- `latest.yml` and `*.blockmap` release assets generated by `electron-builder`.

The updater checks once after startup, downloads an available update, shows download progress, and installs the downloaded update when the user chooses restart. Draft GitHub releases are not visible to the updater.

## GitHub Actions

`.github/workflows/release.yml` publishes releases from tags matching `v*` or manual workflow runs. The workflow installs dependencies with `npm ci` and runs `npm run release` with `GITHUB_TOKEN`.
